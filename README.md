## 1. Мета роботи
Реалізація надійного захисту веб-застосунку шляхом впровадження системи аутентифікації, роботи з сесіями та захисту від поширених веб-вразливостей. Проектування та розробка безпечного REST API для системи моніторингу енергоефективності з використанням механізмів Role-Based Access Control (RBAC) для розмежування прав доступу.

## 2. Технологічний стек
* **Backend:** Node.js, Express.js.
* **База даних:** MongoDB (Mongoose).
* **Безпека:**
    * `Passport.js` (локальна стратегія) для автентифікації.
    * `bcrypt` для хешування паролів.
    * `helmet` для захисту HTTP-заголовків.
    * `express-rate-limit` для захисту від brute-force атак.
    * `express-validator` для санітизації та валідації даних.

## 3. Основний функціонал
* **Реєстрація та вхід:** З обов'язковою валідацією email та довжини пароля.
* **RBAC (Role-Based Access Control):** Розмежування доступу для ролей `energy_manager`, `technologist`, `ceo`.
* **Rate Limiting:** Обмеження кількості спроб входу для захисту від підбору паролів Панель управління, що підлаштовується під мобільні пристрої та відображає дані профілю поточного користувача.
