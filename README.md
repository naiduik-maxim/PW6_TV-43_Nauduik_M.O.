## **1\. Мета роботи**

* Реалізація надійного захисту веб\-застосунку шляхом впровадження системи аутентифікації, роботи з сесіями та захисту від поширених веб\-вразливостей.  
* Проектування та розробка безпечного REST API для системи моніторингу енергоефективності з використанням механізмів Role-Based Access Control (RBAC) для розмежування прав доступу.

## **2\. Технологічний стек**

* **Backend:** Node.js, Express.js.  
* **База даних:** MongoDB (Mongoose).  
* **Безпека:**  
  * Passport.js (локальна стратегія) для автентифікації та управління сесіями.  
  * bcrypt для надійного хешування паролів перед збереженням у БД.  
  * helmet для захисту HTTP-заголовків від поширених атак.  
  * express-rate-limit для захисту від brute-force атак на ендпоінти авторизації.  
  * express-validator для санітизації та валідації вхідних даних.

## **3\. Основний функціонал**

* **Реєстрація та вхід:** Система включає обов'язкову валідацію формату email, довжини пароля та санітизацію імені користувача .  
* **RBAC (Role-Based Access Control):** Жорстке розмежування прав доступу для ролей energy\_manager, technologist та ceo.  
* **Rate Limiting:** Механізм обмеження кількості спроб входу для запобігання автоматизованому підбору паролів.  
* **Адаптивний інтерфейс:** Клієнтська частина реалізована як адаптивна панель управління, що коректно відображається на мобільних пристроях та динамічно виводить дані профілю поточного користувача.

## **4\. Інструкція із запуску**

### **Крок 1: Підготовка оточення**

Переконайтеся, що у вас встановлені **Node.js** та запущені контейнери **MongoDB**.

### **Крок 2: Встановлення залежностей**

Bash  
npm install

### **Крок 3: Налаштування змінних оточення**

Створіть файл .env у корені проекту та додайте необхідні параметри:

Фрагмент коду  
PORT=3000  
MONGO\_URI=mongodb://localhost:27017/energy\_efficiency  
SESSION\_SECRET=ваша\_секретна\_фраза  
NODE\_ENV=development

### **Крок 4: Запуск сервера**

Bash  
npm run dev

Після запуску додаток буде доступний за адресою: **http://localhost:3000/index.html**.

## **5\. Структура проекту**

* **/models** — схеми даних Mongoose для користувачів.  
* **/routes** — логіка REST API та обробка маршрутів автентифікації .  
* **/middleware** — проміжні обробники для перевірки авторизації та ролей (RBAC).  
* **/public** — фронтенд частина додатка: HTML-сторінки, CSS-стилі та клієнтський JavaScript.
